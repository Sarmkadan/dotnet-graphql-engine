# Contributing to dotnet-graphql-engine

First off, thank you for considering contributing to dotnet-graphql-engine! It's people like you that make it such a great tool.

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** the forked repository to your local machine.
3. **Create a branch** for your feature or bug fix (`git checkout -b feature/your-feature-name`).
4. Make your changes and **run tests** to ensure everything works as expected.
5. **Submit a PR** (Pull Request) to the main repository.

## Development Requirements

- **.NET 10.0 SDK**: Ensure you have the latest .NET 10.0 SDK installed on your machine.

## Code Style

- Please follow the existing coding conventions found in the project.
- Provide XML documentation comments for all public APIs.
- **IMPORTANT**: Keep all existing author headers intact in the source files. DO NOT remove them.

## Reporting Issues

If you find a bug or have a feature request, please use GitHub Issues.
When reporting a bug, please include detailed reproduction steps, the expected behavior, and what actually happened.
